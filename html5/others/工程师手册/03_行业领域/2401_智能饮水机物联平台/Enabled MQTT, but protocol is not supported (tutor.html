starcore2022 commented Jul 17, 2023

In the meantime somehow found the setting I was looking for, but couldn't find. It was in 1.3, but not in 1.4. Maybe it is disabled in 1.4. It was the "UA_ENABLE_PUBSUB_MQTT" option, which didn't show up for me previously. For using the cmake terminal command, you just extend the above command with -DUA_ENABLE_PUBSUB_MQTT=ON and it should work.

So, if you don't want to install cmake-curses-gui, than the cmake command is (in 1.3):
cmake -DUA_NAMESPACE_ZERO=REDUCED -DUA_MULTITHREADING=100 -DCMAKE_BUILD_TYPE=Debug -DUA_DEBUG=1 -DCMAKE_C_FLAGS_DEBUG='-fPIC -O0 -g' -DCMAKE_INTERPROCEDURAL_OPTIMIZATION=OFF -DUA_ENABLE_PUBSUB=ON -DENABLE_EXAMPLES=ON -DUA_BUILD_EXAMPLES=ON -DUA_JSON_ENCODING=ON -DUA_ENABLE_MQTT=ON -DUA_FORCE_WERROR=OFF -DUA_ENABLE_PUBSUB_MQTT=ON
